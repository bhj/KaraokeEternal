# Как добавлять песни в Karaoke Eternal

В этом руководстве описан полный процесс добавления песен в библиотеку Karaoke Eternal — от размещения файлов на диске до их появления в интерфейсе.

---

## Поддерживаемые форматы файлов

| Формат | Описание |
|--------|----------|
| **MP3+G** | Аудиофайл `.mp3` (или `.m4a`) плюс файл субтитров `.cdg` с тем же именем. Поддерживается также упакованный вариант `.zip`, содержащий оба файла. |
| **MP4** | Видеофайл с встроенными текстами (поддержка кодеков зависит от браузера плеера). |

---

## Шаг 1. Правильное именование файлов

По умолчанию Karaoke Eternal ожидает, что файлы называются в формате:

```
Исполнитель - Название
```

**Примеры:**

```
Queen - Bohemian Rhapsody.mp3
Queen - Bohemian Rhapsody.cdg

ABBA - Dancing Queen.mp4

The Beatles - Let It Be.zip   ← содержит .mp3 и .cdg
```

> **Важно.** Файлы, имя которых не удалось разобрать, **не появятся** в библиотеке. Проверьте журнал сканера (`scanner.log`) или вывод консоли, если песня не отображается.

---

## Шаг 2. Разместите файлы в папке

Скопируйте (или примонтируйте) папку с песнями туда, где её может прочитать сервер:

- **Docker** — смонтируйте папку в контейнер (например, `/mnt/karaoke`):

  ```yaml
  volumes:
    - /путь/к/моим/песням:/mnt/karaoke
  ```

- **npm** — используйте любой локально доступный каталог, например `~/Музыка/Karaoke`.

Допустимы вложенные подпапки — сканер просматривает каталоги рекурсивно.

---

## Шаг 3. Добавьте папку в приложении

1. Откройте Karaoke Eternal в браузере и войдите под учётной записью **администратора**.
2. Перейдите в **Account** (значок в нижней панели навигации).
3. В разделе **Preferences** выберите **Media Folders**.
4. Нажмите **Add folder** и укажите путь к папке с песнями:
   - Docker: `/mnt/karaoke`
   - npm: полный путь к каталогу (например, `/home/user/karaoke`)
5. Нажмите **Save**. Сканер запустится автоматически.

После окончания сканирования песни появятся в разделе **Library**.

---

## Шаг 4. Проверьте результат

- Откройте вкладку **Library** — вы увидите список исполнителей и их треки.
- Если песни отсутствуют, убедитесь, что:
  - Формат файла поддерживается (MP3+G или MP4).
  - Имена файлов соответствуют шаблону `Исполнитель - Название`.
  - Путь к папке указан верно.
  - Сканер завершил работу (следите за консолью или `scanner.log`).

---

## Дополнительно: настройка формата имён файлов

Если ваши файлы названы иначе (например, `Название - Исполнитель`), создайте в папке файл `_kes.v2.json`:

```json
{
  "artistOnLeft": false
}
```

Полный список настроек по умолчанию:

```js
{
  // нормализация артикля (A, An, The); false — отключить
  articles: ['A', 'An', 'The'],

  // true  → «Исполнитель - Название»
  // false → «Название - Исполнитель»
  artistOnLeft: true,

  // разделитель между исполнителем и названием
  delimiter: '-',
}
```

Файл `_kes.v2.json` применяется к текущей папке и всем её подпапкам. Если подпапка содержит собственный `_kes.v2.json`, он имеет приоритет.

## Дополнительно: шаблоны полей метаданных

Для тонкой настройки можно переопределить любое поле метаданных с помощью шаблонов [JSON-e](https://json-e.js.org):

```js
// Установить исполнителя напрямую
{
  "artist": "Имя исполнителя"
}
```

```js
// Удалить слово «Karaoke» из названия
{
  "title": {
    "$eval": "replace(title, \"Karaoke\", \"\")"
  }
}
```

```js
// Взять исполнителя и название из тегов файла
{
  "artist": "${meta.artist}",
  "title": "${meta.title}"
}
```

Доступные поля шаблона: `artist`, `artistNorm`, `title`, `titleNorm`, `name` (имя файла без расширения), `meta` (теги файла), `dir` (путь к папке), `dirSep` (разделитель пути).

## Ручной запуск сканера

Можно запустить сканер вручную из командной строки:

```bash
# Сканировать все папки
karaoke-eternal-server --scan all

# Сканировать конкретные папки (по ID)
karaoke-eternal-server --scan 1,2
```
